Javascript Plugins

Each javascript widget has an associated HTML file, Javascript file and a CSS file. There may also be images and third party JS libraries as well. The code is written to be easily added to each project. These plugins are usually called from a jQuery $(document).ready(function() which waits for the DOM to be fully loaded, and then event handling is attached to these widget elements. Note: there should only ever be 1 document.ready function for the document. There will however be times when you need to call a plugin at some time after the document is loaded. e.g. After an AJAX response. In these cases the plugin code will be inserted into the function that handles the AJAX response. The javascript should always be unobtrusive. This means that the javascript controlling the page interactions should be seperated from the HTML and never placed inside the HTML file, much like CSS should never be placed directly inside the HTML page, rather it should be in a separate file.

The HTML

The entire HTML content of the plugin is contained inside a div with the class 'module'. On the demo pages where there are more than one plugin, there will be a different 'module' div for each instance of that plugin. Simply copy the contents of this div and place it into the project where you need it.

The CSS

Each plugin may have its own css file. When integrating the plugin into your project, it is purely a matter of copying the contents of this CSS file and pasting into skin.css in your project. There is a place maked for the plugins in this file towards the end, before the media queries. Each section of this file is commented, like this: "/* ========== Plugin CSS ==================== */" so it will be easy to find.

The Javascript

Each plugin has it's own JS file which you will need to add to the project. This file should be placed in the plugins folder - '/public/resources/js/plugins'. The link to this file should be called in the HTML after the jQuery library and before the $(document).ready() function which is calling it. In the demo page the document.ready function is placed inside a script tag in the HTML. However this is only for convenience and should not be done when integrating it into a project.

There is a file called interface.js which holds the project's document.ready function, this is located in '/public/resources/js/', and it is from here that the plugins should be called.

Example of using the Googlemap Plugin:

This example shows how you would integrate the Googlemap plugin into a project. The first thing you need to do is get all the external resources for the plugin. Get the files and put them in the appropriate sub-directory in the js folder. These are the scripts:

<!-- googlemap code -->
<script language="javascript" type="text/javascript" src="http://maps.googleapis.com/maps/api/js?sensor=false"></script>
<script language="javascript" type="text/javascript" src="js/libs/markercluster.js"></script>
<script language="javascript" type="text/javascript" src="js/plugins/map1.0.js"></script>

You will notice that the markercluster.js file is located in the libs folder. This is because it is a third party library. A plugin may from time to time require these e.g. jQuery UI library, and all third party JS is located here. Also, there are some images associated with this plugin (see the plugin code below). You can that in the options for this plugin there are two images: "img/pin/pin.png" and "img/pin/pin-shadow.png". These two assets will need to be copied across too.

In the googlemap.html file is this markup:

<div class="module">
    <h2>Googlemap Demo</h2>
    <div id="mapDiv"></div> <!-- end of mapDiv -->
</div> <!-- end of module -->

Now, all the code for the plugin is in the module div. We may not need the <h2> tag, so we will only copy <div id="mapDiv"></div> and paste it into our project page where we will need it. Next, open up the CSS file and copy the code across to skin.css into the plugin section. This is towards the bottom before the media queries and is thus commented: "/* ========== Plugin CSS ==================== */". This is the code:

#mapDiv {width:100%;height:500px}

The last thing you need to do is get the javascript that initialises the plugin and put it in the appropriate place in the interface.js file. Next, lets look at the interface.js file and see where to put this code. Here is this file:

// custom javascript functions

// this object to be called whatever the project name is
Project = {
    init : function()
    {
        // call plugins from here
    }
}

$(document).ready(function()
{
    // initialise the Project object
    Project.init();
});

You will notice two things about this structure. One, there is a namespace created for the project. In this case it is Project. This object has an 'init' function that calls its other methods. Two, this init function is called from a $(document).ready(). This should be the only place in the project that a document.ready function is being called. Now I will create a new function called makeMap() and use it to initialise the plugin from inside the init() function. Then I will copy the plugin code from the HTML file and put it in the makeMap() function. Inside makeMap() function I will test to see if the map element exists. This is so the plugin code won't run on any pages that doesn't have the div #mapDiv. You can do this using the "length" property like so: if ($("#mapDiv").length > 0) {plugin code here}

This is what interface.js will look like with the plugin code:
